# Test Specification: Issue Session Summary

## Test Environment

- **Framework:** pytest (with pytest-asyncio for async tests)
- **Mocking:** unittest.mock for `PlatformProtocol` and subprocess calls
- **Fixtures:** Temporary directories with synthetic `prd.md` and
  `tasks.md` files

## Unit Tests

### TS-108-1: parse_source_url returns SourceIssue for GitHub URL

**Requirement:** 108-REQ-1.1, 108-REQ-1.2

**Setup:**
- Create a `prd.md` file with content including:
  ```
  ## Source

  Source: https://github.com/agent-fox-dev/agent-fox/issues/359
  ```

**Action:** Call `parse_source_url(prd_path)`.

**Assertions:**
- Returns a `SourceIssue` (not `None`)
- `result.forge == "github"`
- `result.owner == "agent-fox-dev"`
- `result.repo == "agent-fox"`
- `result.issue_number == 359`

### TS-108-8: post_issue_summaries posts comment for completed spec

**Requirement:** 108-REQ-4.1, 108-REQ-2.1, 108-REQ-2.2

**Setup:**
- Create spec directory with `prd.md` (GitHub source URL) and
  `tasks.md` (all groups `[x]`)
- Mock `PlatformProtocol` with `add_issue_comment` as async mock
- `completed_specs = {"108_my_spec"}`
- `already_posted = set()`
- Mock `git rev-parse develop` to return `"abc123"`

**Action:** Call `await post_issue_summaries(platform, specs_dir,
completed_specs, already_posted, repo_root)`.

**Assertions:**
- `platform.add_issue_comment` called once with the issue number
  and a body containing the spec name
- Return value includes `"108_my_spec"`

## Integration Tests

### TS-108-SMOKE-1: End-to-end issue summary posting

**Requirement:** 108-REQ-4.1, 108-REQ-4.2

**Setup:**
- Create a temp spec directory with:
  - `prd.md` with `Source: https://github.com/test/repo/issues/1`
  - `tasks.md` with one completed task group
- Mock `PlatformProtocol` with async `add_issue_comment`
- Construct an `Orchestrator` with the mock platform
- Stub the task graph so the spec is already completed

**Action:** Run the orchestrator (which should detect the spec as
completed and post the summary).

**Assertions:**
- `add_issue_comment(1, body)` called exactly once
- `body` contains the spec name
- `body` contains a commit SHA (or `"unknown"`)
- `body` contains `*Auto-generated by agent-fox.*`
