# agent-fox Configuration Reference

This document lists every configuration option supported by agent-fox.
The simplified `config.toml` generated by `agent-fox init` contains only
the most commonly changed settings. Add any section below manually to
`config.toml` to override the defaults.

| Field | Type | Default | Bounds | Description |
|-------|------|---------|--------|-------------|
| `parallel` | int | `2` | 1ŌĆō8 | Maximum number of parallel coding sessions |

| `sync_interval` | int | `5` | Ōēź 0 | Task-group sync interval in number of sessions |
| `hot_load` | bool | `true` | ŌĆö | Hot-reload spec files between sessions without restarting the orchestrator |
| `max_retries` | int | `2` | Ōēź 0 | Maximum number of automatic retries per task group |
| `session_timeout` | int | `30` | Ōēź 1 | Per-session timeout in minutes |
| `inter_session_delay` | int | `3` | Ōēź 0 | Delay in seconds between consecutive session launches |
| `max_cost` | float\|null | `null` | ŌĆö | Hard cost ceiling for the entire run (null = no limit) |
| `max_sessions` | int\|null | `null` | ŌĆö | Maximum total sessions in a run (null = no limit) |
| `audit_retention_runs` | int | `20` | Ōēź 1 | Number of run audit logs to retain on disk |

**Example:**


## routing

Adaptive model routing configuration. Controls when the orchestrator escalates
to a more capable model tier based on past session outcomes.

> **Note:** This is a hidden section ŌĆö it does not appear in the simplified
> template. Add it manually when you want to tune routing behaviour.

| Field | Type | Default | Bounds | Description |
|-------|------|---------|--------|-------------|
| `retries_before_escalation` | int | `1` | 0ŌĆō3 | Failed retries before escalating to the next model tier |
| `training_threshold` | int | `20` | 5ŌĆō1000 | Minimum outcome records needed before routing model training |
| `accuracy_threshold` | float | `0.75` | 0.5ŌĆō1.0 | Minimum routing accuracy to trust the trained model |
| `retrain_interval` | int | `10` | 5ŌĆō100 | Number of new outcomes between routing model retrains |
| `max_timeout_retries` | int | `2` | Ōēź 0 | Maximum timeout retries before falling through to escalation (0 = disable timeout handling) |
| `timeout_multiplier` | float | `1.5` | Ōēź 1.0 | Factor by which `max_turns` and `session_timeout` are extended on each timeout retry |
| `timeout_ceiling_factor` | float | `2.0` | Ōēź 1.0 | Maximum `session_timeout` as a multiple of the original configured value |

**Example:**

```toml
[routing]
retries_before_escalation = 2
training_threshold = 50
accuracy_threshold = 0.80
retrain_interval = 20
max_timeout_retries = 3
timeout_multiplier = 1.5
timeout_ceiling_factor = 2.0
```

## knowledge

Knowledge store and semantic search configuration. The knowledge store
persists facts learned across sessions for use as context in future sessions.

> **Note:** This is a hidden section.

| Field | Type | Default | Bounds | Description |
|-------|------|---------|--------|-------------|
| `store_path` | str | `".agent-fox/knowledge.duckdb"` | ŌĆö | Path to the DuckDB knowledge store file |
| `embedding_model` | str | `"all-MiniLM-L6-v2"` | ŌĆö | Sentence-transformers model used for fact embeddings |
| `embedding_dimensions` | int | `384` | ŌĆö | Dimensionality of embedding vectors |
| `ask_top_k` | int | `20` | Ōēź 1 | Number of top-ranked facts returned by knowledge queries |
| `ask_synthesis_model` | str | `"STANDARD"` | ŌĆö | Model tier used to synthesise query answers |
| `confidence_threshold` | float | `0.5` | 0.0ŌĆō1.0 | Minimum similarity score for fact inclusion in session context |
| `fact_cache_enabled` | bool | `true` | ŌĆö | Pre-compute fact rankings at plan time for faster session start |
| `dedup_similarity_threshold` | float | `0.92` | ŌĆö | Cosine similarity threshold for near-duplicate detection |
| `contradiction_similarity_threshold` | float | `0.8` | ŌĆö | Cosine similarity threshold for contradiction candidates |
| `contradiction_model` | str | `"SIMPLE"` | ŌĆö | Model tier for contradiction classification LLM calls |
| `decay_half_life_days` | float | `90.0` | ŌĆö | Days for fact confidence to halve |
| `decay_floor` | float | `0.1` | ŌĆö | Effective confidence below which facts are auto-superseded |
| `cleanup_fact_threshold` | int | `500` | ŌĆö | Active fact count above which decay cleanup runs |
| `cleanup_enabled` | bool | `true` | ŌĆö | Enable/disable end-of-run fact lifecycle cleanup |
| `retrieval` | table | ŌĆö | ŌĆö | Adaptive retrieval tuning parameters (see `[knowledge.retrieval]`) |

### knowledge.retrieval

Tuning parameters for the unified adaptive retriever (spec 104). Controls
multi-signal RRF fusion, token budgeting, and per-signal candidate caps.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `rrf_k` | int | `60` | RRF smoothing constant (denominator offset) |
| `max_facts` | int | `50` | Maximum facts in the fused anchor set |
| `token_budget` | int | `30000` | Maximum characters for the formatted context block |
| `keyword_top_k` | int | `100` | Candidate cap for the keyword signal |
| `vector_top_k` | int | `50` | Candidate cap for the vector signal |
| `entity_max_depth` | int | `2` | Max BFS traversal depth for entity signal |
| `entity_max_entities` | int | `50` | Max entities traversed in entity signal |
| `causal_max_depth` | int | `3` | Max traversal depth for causal signal |

**Example:**

```toml
[knowledge]
ask_top_k = 30
confidence_threshold = 0.6
fact_cache_enabled = true

[knowledge.retrieval]

```

## archetypes

Archetype enable/disable toggles and per-archetype advanced configuration.

After the reviewer consolidation, the former `skeptic`, `oracle`, and `auditor`
archetypes are unified into a single `reviewer` archetype with mode-based
behaviour (`pre-review`, `drift-review`, `audit-review`, `fix-review`).

| Field | Type | Default | Bounds | Description |
|-------|------|---------|--------|-------------|
| `coder` | bool | `true` | ŌĆö | Enable the Coder archetype |
| `reviewer` | bool | `true` | ŌĆö | Enable the Reviewer archetype (replaces skeptic, oracle, auditor) |
| `verifier` | bool | `true` | ŌĆö | Enable the Verifier archetype (post-code correctness checks) |
| `instances` | table | see below | ŌĆö | Per-archetype instance counts |
| `reviewer_config` | table | see below | ŌĆö | Reviewer-specific configuration (replaces skeptic_config, oracle_settings, auditor_config) |
| `models` | dict[str, str] | `{}` | ŌĆö | Per-archetype model tier overrides (e.g. `{coder = "STANDARD"}`) |
| `allowlists` | dict[str, list[str]] | `{}` | ŌĆö | Per-archetype extra bash command allowlists |
| `max_turns` | dict[str, int] | `{}` | ŌĆö | Per-archetype maximum turn limits (Ōēź 0) |
| `thinking` | dict[str, ThinkingConfig] | `{}` | ŌĆö | Per-archetype extended thinking configuration |
| `overrides` | dict[str, PerArchetypeConfig] | `{}` | ŌĆö | Per-archetype config overrides (model_tier, max_turns, allowlist, modes) |
| `custom` | list[CustomArchetypeConfig] | `[]` | ŌĆö | Custom archetype definitions |

**Example:**

```toml
[archetypes]
reviewer = true
verifier = true
# Override model tiers per archetype
models = {coder = "STANDARD", reviewer = "ADVANCED"}

# Limit turns for specific archetypes
max_turns = {coder = 100, verifier = 50}
```

## pricing

Custom per-model pricing for cost tracking. By default agent-fox ships with
pricing for the standard Claude models. Override here when using custom
deployments or when Anthropic updates prices.

> **Note:** This is a hidden section.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `models` | dict[str, ModelPricing] | (see below) | Per-model pricing entries |

Each entry in `models` is a TOML inline table or sub-table with these fields:

| Sub-field | Type | Default | Bounds | Description |
|-----------|------|---------|--------|-------------|
| `input_price_per_m` | float | `0.0` | Ōēź 0 | USD per million input tokens |
| `output_price_per_m` | float | `0.0` | Ōēź 0 | USD per million output tokens |
| `cache_read_price_per_m` | float | `0.0` | Ōēź 0 | USD per million cache-read input tokens |
| `cache_creation_price_per_m` | float | `0.0` | Ōēź 0 | USD per million cache-creation input tokens |

**Built-in defaults:**

| Model | Input $/M | Output $/M | Cache read $/M | Cache creation $/M |
|-------|-----------|------------|----------------|--------------------|
| `claude-haiku-4-5` | 1.00 | 5.00 | 0.10 | 1.25 |
| `claude-sonnet-4-6` | 3.00 | 15.00 | 0.30 | 3.75 |
| `claude-opus-4-5` | 5.00 | 25.00 | 0.50 | 6.25 |
| `claude-opus-4-6` | 5.00 | 25.00 | 0.50 | 6.25 |

**Example (override a single model's pricing):**

```toml
[pricing.models.claude-sonnet-4-6]
input_price_per_m = 3.00
output_price_per_m = 15.00
cache_read_price_per_m = 0.30
cache_creation_price_per_m = 3.75
```

## caching

Prompt caching configuration. Controls whether `cache_control` markers are
injected into Anthropic API requests, reducing input token costs on cache hits.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `cache_policy` | str | `"DEFAULT"` | Caching strategy: `NONE`, `DEFAULT` (5-min TTL), or `EXTENDED` (1-hour TTL) |

**Policies:**

| Policy | `cache_control` marker | TTL | Cost trade-off |
|--------|------------------------|-----|----------------|
| `NONE` | None | ŌĆö | No caching ŌĆö identical behaviour to pre-caching releases |
| `DEFAULT` | `{"type": "ephemeral"}` | 5 minutes | Reduces input costs on repeated calls within a short window |
| `EXTENDED` | `{"type": "ephemeral", "ttl": "1h"}` | 1 hour | Higher cache-write cost; pays off on long-running sessions |

**Token threshold:** Caching is automatically skipped when the system prompt
is estimated to be below the model's minimum cacheable size (Ōēł2 048 tokens
for Sonnet-class models, Ōēł4 096 for Opus/Haiku-class). Prompts below this
threshold are passed through unchanged regardless of policy.

**Example:**

```toml
[caching]
cache_policy = "DEFAULT"   # NONE | DEFAULT | EXTENDED
```

**Rollback:** Set `cache_policy = "NONE"` to fully disable caching with no
code changes required.
